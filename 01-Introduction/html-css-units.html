HTML & CSS Units

Numbers, Lengths and Percentages:

There are various numeric value types used in CSS. It's classified into four types.

Data type Description
<integer> An <integer> is a whole number such as 100 or -1024
<number> A <number> represents a decimal number - it may or may not have a fractional part. For example, 0.255, 128, or -1.3.
<dimension> A <dimension> is a <number> with a unit attached to it. For example, 50px, 3s, or 30deg. It's an umbrella category that includes the <length>, <time>, <angle>, and <resolution> types.
<percentage> A <percentage> represents a fraction of some other value. For example, 50%. These values are always relative to another quantity.

Absolute Units

Those units which are always the same in any context. px is the only absolute unit that you should use in projects. The rest make more sense in print settings as they are related to physical units such as in (inch) and cm (centimeter).

Unit Name Equivalent to
cm Centimeters 1cm = 37.8px = 25.2/64in
mm Milimeters 1mm = 1/10th of 1cm
in Inches 1in = 2.54cm = 96px
pc Picas 1pc = 1/6th of 1in
px Pixels 1px = 1/96th of 1in

Note: 1px doesn't necessarily equal one physical device pixel. On HD displays, it may span multiple phusical pixels. Similarly, 1cm in CSS often doesn't corresponds to SI meter.

On a large TV scree, it typically is longer than that. The lengths are perceptual: 16px looks roughly the same on a phone, laptop or TV screen at typical viewing distance.

Relative Units

Units that can change based on their context. There are many that you are likely to encounter and want to use.

1. em and rem:

em and rem both refer to a font size, though they are often used to define other sizes. You'll see both of them being used often but as a rule-of-thumb, prefer rem.

  • 1em is the font-size of an element (or element's parent if you're using it to set font-size). So for example, if an element's font-size is 16px then setting it's width to 4em would make its width 64px(4 * 16 = 64).
  • 1rem is the font-size of the root element (:root or html). The math works the same as it did with em, but without the added complexity of keeping track of the parent's font size. Relying on em could mean that a particular size could change if the context changes, which is very likely not the behaviour you want.

2. viewport units:

The units vh and vw relate to the size of the viewport. Specifically 1vh is equal to 1% of the viewport height and 1vw is equal to 1% of viewport width. Useful when you want something to be sized relative to the viewport, examples include full-height heroes and full-screen app-like interfaces.

Exploring an example:

In the example below, you can see how some relative and absolute lengths units behave.

  1. The first box has a width set in pixels. As an absolute unit, this width will remain the same no matter what else changes.
  2. The second box has a width set in vw (viewport width) units. It's a relative unit, so 40vw is 20 percent of the width of the viewport. If you change the width of your browser window, the size of the box will change accordingly.
  3. The third box uses em units. These are relative to the parent element's font-size. It's currently set as size of 1em on the containing <div>. Change this value to 1.5em and you will see that the font-size and the width of third container will change.
