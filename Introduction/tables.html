The scope attribute can be added to the th element to tell screen readers exactly what the header is a header for - is it a header for a row it is in, or the column.

For headers spanning multiple columns or rows we use scope="colgroup" or scope="rowgroup" respectively.

The id & header attributes

How it works?

  1. Add a unique id to each th.
  2. For subheaders, add headers pointing to the main heading’s id.
  3. For each td, add headers with the related row and column ids (space-separated).

Note: This method creates very precise associations between headers and data cells but it uses a lot more markup and does not leave any room for errors. The scope approach is usually sufficient for most tables.
